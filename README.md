# 公共建筑碳排放计算报告系统 - 精细化版 V3.0

基于 GB/T 51366-2019《建筑碳排放计算标准》和 JS/T 303-2026《公共机构碳排放核算指南》

将建筑碳排放计算拆分为 **39个独立计算单元**，每个单元专注于一个具体的计算任务，实现单一职责、独立完整、易于扩展和维护。

---

## 📁 项目结构

```
碳排放报告模板/
├── main.py                       # 统一入口脚本
├── generate_granular.py          # 精细化Excel生成器（39个工作表）
├── sync.py                       # OpenXML Excel-Word同步模块
├── example_sync.py               # 同步使用示例
├── requirements.txt              # Python依赖
├── README.md                     # 本说明文档
└── 公共建筑碳排放计算表_精细化版.xlsx  # 生成的精细化Excel（运行后生成）
```

## 📊 精细化计算单元（39个工作表）

### 设计理念

| 原则 | 说明 | 优势 |
|-----|------|------|
| **单一职责** | 每个工作表只计算一个具体内容 | 逻辑清晰，易于理解 |
| **独立完整** | 每个工作表都是自包含的（输入→公式→输出） | 可独立使用 |
| **易于扩展** | 新增计算单元只需添加一个独立工作表 | 不影响现有功能 |
| **易于维护** | 修改某个计算不影响其他计算单元 | 降低维护成本 |

## 🔗 Excel-Word 数据同步

### 使用命令行同步

```bash
# 基础同步
python main.py --sync

# 指定输出文件
python main.py --sync --output 我的报告.docx
```

### 使用Python代码同步

```python
from sync import SyncManager, SyncConfig, CellMapping

# 创建配置
config = SyncConfig(default_sheet_name="项目基本信息")
config.mappings = [
    CellMapping("D4", "项目名称"),
    CellMapping("D7", "建筑面积"),
    CellMapping("D18", "年用电量"),
]

# 执行同步
sync = SyncManager(config)
sync.load_excel("公共建筑碳排放计算表_精细化版.xlsx")
sync.load_word("公共建筑碳排放计算报告.docx")
sync.sync_excel_to_word("输出报告.docx")
```

### 运行示例

```bash
python example_sync.py
```

## 🔧 扩展开发

### 新增计算单元

1. 在 `generate_granular.py` 中添加新的工作表生成函数：

```python
def create_39_new_unit(wb):
    """39. 新功能"""
    ws = create_sheet(wb, "39. 新功能", 'FF5722',
                     "新功能计算", 22)

    row = 3
    row = add_input_section(ws, row, "新参数",
        ["参数", "数值", "单位", "来源", "说明"],
        [
            ["参数1", "", "-", "", ""],
        ], 0)

    row += 1
    add_formula_row(ws, row, "计算结果",
                   "=B3*B4", "单位", "说明")
```

2. 在 `main()` 函数中注册：

```python
sheets = [
    # ... 现有工作表
    ("39. 新功能", create_39_new_unit),
]
```

3. 重新生成：

```bash
python main.py --regenerate
```

### 修改计算公式

每个工作表都是独立的，修改某个表不会影响其他表。

## 🆚 精细化版 vs 综合版

| 特性 | 精细化版（39表） | 综合版（13表） |
|-----|---------------|--------------|
| 工作表数量 | 39个 | 13个 |
| 每个表职责 | 单一功能 | 多个功能 |
| 修改影响范围 | 只影响本单元 | 影响多个功能 |
| 新增功能 | 添加新工作表 | 需修改现有表 |
| 学习成本 | 低（每表专注一个功能） | 高（表内容多） |
| 数据引用 | 显式跨表引用 | 隐含在同一表 |
| 适用场景 | 大型项目、团队协作 | 小型项目、快速使用 |
